# /learn — 学びを CLAUDE.md に永続化するコマンド

ユーザーが「これは覚えておいて」「同じミスをしないように」と言った内容を、CLAUDE.md の **学習履歴** セクションに追記するのがあなたの仕事です。

## 引数

ARGUMENTS には学びの内容が渡されます（自由記述）。

## 実行手順

1. `CLAUDE.md` を読む
2. 「学習履歴」セクション（`## 📚 学習履歴` 見出し）を見つける
3. 今日の日付（ISO 形式）の見出しが既にあれば、その下に項目を追加。なければ新しい日付見出しを作る
4. 以下のフォーマットで追記:

```markdown
### YYYY-MM-DD
- [ ] [学び本文]
  - **なぜ**: [根本原因や背景]
  - **どうする**: [次回どう振る舞うか]
```

5. ユーザーに「CLAUDE.md に追記しました」と短く伝える

## ルール

- **重複を作らない**: 既存の項目と同じ趣旨なら、新規追加せずに既存を更新する旨をユーザーに伝える
- **抽象化しすぎない**: 「丁寧に書く」ではなく「JS 文字列の中で英語引用を使うときは外側のクォートと衝突しないよう日本語の鉤括弧『』を使う」のように再現可能な粒度で書く
- **責任転嫁しない**: 「ユーザーが〜と言ったから」ではなく「次に同じ状況で何をするか」を書く
- **3 行以内**で 1 学びを書く（長くなるなら 2 つに分ける）

## 例

ユーザー: `/learn JS の文字列で英語引用を入れるなら鉤括弧を使う`

追記内容:
```markdown
### 2026-05-16
- [ ] JS の文字列リテラルで英語引用を入れる場合、ダブルクォート同士の衝突で SyntaxError になる
  - **なぜ**: `"...\"foo\"..."` の `\"` を忘れると外側の文字列がそこで終わってしまう
  - **どうする**: 英語引用は日本語の鉤括弧「」で囲む、または外側をシングルクォートにする
```
