# verify — 自己検証スキル

実装が完了したと思った瞬間に必ず実行する。「動くはず」で止めない。

## 手順

### 1. テストを走らせる
プロジェクトに応じて以下のどれかを走らせる:
- `npm test`
- `pytest`
- `cargo test`
- `go test ./...`

テストが存在しない場合は「テスト未整備」と明示してスキップ。

### 2. UI の検証（Web の場合）
Chrome DevTools MCP を使って:
1. `new_page` でローカルサーバーまたは file:// を開く
2. `take_screenshot` で full page を撮る
3. `list_console_messages` でエラー 0 を確認
4. 主要ボタン・フォームを `click` / `fill` で 1 つだけ試す

### 3. CLI の検証（CLI ツールの場合）
- 想定する代表的な引数で実行
- exit code を確認
- 標準出力が想定通りか確認

### 4. 結果サマリ
以下の形式でユーザーに報告:

```
## 検証結果

- テスト: PASS (12/12) または FAIL (詳細)
- UI: スクショ OK / コンソールエラー 0
- 主要動作: 確認済み（[操作内容]）
- 残課題: なし or [箇条書き]
```

## 重要

- **「動くはず」と書かない。** 動いたか動かなかったかを書く。
- **失敗を隠さない。** 1 つでも失敗があれば PASS と書かない。
- **テスト不能なものは「テスト不能 — X が必要」と明示する。**
